# 0004 — 领域记忆:记结构,不固化数值结论

- 状态: Accepted (2026-06-14)

## 背景

阶段二自进化以 L1 领域记忆为重心。数据分析有个别的领域没有的陷阱:**数据会变**。
朴素的"缓存历史结论"(如 `insight: 留存率=12%`)在数据文件更新后会**过时甚至误导**——
模型据此分析新数据会得出错误判断。

## 决策

记忆只沉淀**稳定的领域知识**,坚决不固化**易变的数值结论**。具体四类:

1. **数据集画像 dataset_profile** — 拆**结构层**(schema/列语义,稳定)与**统计层**
   (分布/质量数值,易变);用**列名集合指纹**判定失效:指纹不变→结构层延续、仅统计层
   标 stale 待重算;增删列→整条作废重探(`memory/profiler.py`)。
2. **口径定义 metric_definition** — 业务语义(「活跃用户=过去7天登录」),高危子类,
   自动抽草案 + "下次使用时模型复述、用户不纠正即确认"的轻确认闭环。
3. **分析手法偏好 analysis_pref** — 惯用角度/图表风格。
4. **待复核疑点 open_concern** — 数据质量坑。

**明确不记**:具体数值结论(finding,如「留存率=12%」)。

## 理由

数据分析的"好结论"绑定具体数据快照;数据一变即失效。把"结构/口径/手法"(跨数据稳定)
与"结论数值"(随数据漂移)分开,只记前者,换取"记忆永不因过时数据误导分析"。这条红线
同时决定了画像的分层失效设计与 eval 的"验证方法非数值"原则(见 ADR 0005)。

## 影响

新增 `memory/`(model/store/profiler/injector);agent_loop 经 `memory_injector`/
`memory_recorder` 回调注入,不反向依赖 memory。画像确定性生成(零 LLM、零误抽)。
检索用关键词+中文子串(几十到几百条量级,不引向量库——过度设计)。
